## NAVIGATION & STATE RULES

- Use a local step state: const [step, setStep] = useState(0)
- Steps: 0=splash, 1=problemScreen, 2=shiftScreen, 3=promiseScreen, 4=focusAreas, 5=duration, 6=dailyGoal, 7=appSelection, 8=permissions, 9=completion
- Navigation between steps: always use a goNext() function that increments step with a smooth cross-fade transition using Animated (opacity fade 200ms between steps)
- NO back button on any screen. This is a forward-only flow.
- On the completion screen CTA press:
  - Call updateSettings with all collected data: focusBodyAreas, preferredDuration (mapped: "20"→20, "45"→45, "60"→60), dailyGoal, lockedApps (array of app ids), hasCompletedOnboarding: true
  - Then navigate to /(tabs)/ using router.replace('/(tabs)/')

## TRANSITION BETWEEN STEPS
Between every step change, do a quick opacity fade:
- Fade current step out (opacity 1 → 0, 150ms)
- Switch step state
- Fade new step in (opacity 0 → 1, 200ms)
Use a single Animated.Value for this, wrapping the whole step content area.
